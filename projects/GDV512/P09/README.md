## Generic Description

Use branching, commits, reviews, issue tracking, and build notes; avoid integration surprises; document individual contributions.

## Specific Execution

In pairs or small teams, build a game with at least 30 meaningful commits, issue tracking, and a final merge review. Each student must own a visible technical or content feature. The final repository must include release notes and known issues.

## Grading Rubric

25 pts version-control hygiene; 25 pts working integrated game; 20 pts contribution clarity; 15 pts issue tracking/build review; 15 pts team retrospective.
