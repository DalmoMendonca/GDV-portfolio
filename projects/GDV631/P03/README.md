## Generic Description

Use telemetry responsibly; communicate data collection; turn metrics into design insight; distinguish useful instrumentation from surveillance.

## Specific Execution

Build a game that collects local or simulated analytics such as deaths, retries, choices, fail locations, input errors, retention time, or settings changes. Include consent language, an event schema, a dashboard or report view, and an explanation of what the data suggests.

## Grading Rubric

30 pts ethical data handling; 20 pts useful analytics/schema; 20 pts design insight; 15 pts transparency; 15 pts documentation.
